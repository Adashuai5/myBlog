windows 7与linux双系统安装（后续）

发表于 2018-05-27 |
字数统计: 749

前两天美滋滋得安装了windows 7下的linux双系统ubuntu，详情见 链接 ，以为从此即将开始愉快的学（zhuang）习（bi）之旅了。没想到接下来就是各种强行熟悉其各种命令的噩梦。。。

首先是各种无法启动

于是我选择各种重装，结果悲剧了。

每次重启都是这个，崩溃

于是跟着大神们开始启动救援模式，具体可自行google“grub rescue”这篇是我写文章时打开的 博客 链接

当我跟着大神们一步步

终于成功进入grub2界面，即可选择系统（每个人不一样，可以多看看不同博客，有细微差别，我当时尝试了不同格式上述命令，终于属于找到自己电脑的，当你都看完你就都懂了）

此时问题又来了，我进不了ubuntu啊，因为我就是启动不了才发生后续问题的，这个问题至今未解决。。那怎么办，无法更新grub就一直会重启出现grub rescue啊（鬼知道我打了多少遍上面的命令，相信你懂得）我找了各种解决办法，包括什么用尝试启动ubuntu来下载 引导修复（boot repair） ，但是我ubuntu无法联网啊，U盘也没有（上一篇免U盘装双系统），还有什么live cd，都没有不会啊 （上述情况任何一样有都有相应解决方法，可视情况选择，具体我给的链接应该涵盖了）

在我都打算用grub rescue 作为引导启动项的时候（每次开机打一遍上面的命令），我发现了新的简便方法，只需用到 EasyBCD（装系统时用到）。由于我们可以进入grub2界面进入windows系统，因此操作方便（其实之前一直想尝试用 DiskGenius参考博客重建MBR的方法，但是怕windows系统也崩溃。。其实一直最担心的是这个，好多数据的。。结果担心是多余）

了解MBR GRUB 可参考博客 链接

通过EasyBCD 重建MBR 解决grub rescue的问题具体可参考 博客 链接

当然我的ubuntu还没删，万一哪天高兴打算再研究一下，估计不行的话会删尝试虚拟机的方法，不过可能还是用笔记本破釜沉舟直接安装linux较好，有空在试吧。

本次装双系统花了2个下午加3个晚上，终于成功回到了起点。。。
